# DooHub — Claude Instructions

## Project Structure

This is a monorepo containing:
- `app/` — Flutter mobile app
- `web/` — Next.js frontend
- `backend/` — FastAPI backend

## Response Style

Use markdown formatting, code blocks, and detailed explanations as appropriate. This project is NOT a Telegram bot — do not restrict output formatting.

## GitHub Issues

- Always assign at least one label when creating issues
- Create **separate issues** for each part of the monorepo (app, web, backend) — never combine them into one issue
- Available labels:
  - `app` — Flutter mobile app
  - `web` — Next.js frontend
  - `backend` — FastAPI backend
  - `general` — Something general for this project
